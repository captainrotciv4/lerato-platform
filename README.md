## What this is

A single web + mobile application that serves three organizations as branded portals, with shared data (donors, partners, joint events) managed once and surfaced cleanly in each org's view. Replaces spreadsheets, WhatsApp threads, and paper records across People · Programs · Operations · Impact.

## Tech stack

| Layer | Choice | Why |
|---|---|---|
| Framework | **Next.js 15** (App Router) | Single codebase for web + API. Server Actions cut boilerplate. |
| Language | **TypeScript** | Compile-time safety for donor & beneficiary data. |
| Database | **PostgreSQL 16** | Row-level security for multi-tenant isolation. |
| ORM | **Prisma 5** | Best multi-tenant story. Type-safe queries. |
| Auth | **Auth.js v5** | Email + phone OTP, session management, RBAC hooks. |
| UI | **Tailwind CSS + Shadcn/ui** | Fast theming across three branded portals. |
| Forms | **React Hook Form + Zod** | Schema-validated forms; same Zod schemas reused server-side. |
| Data fetching | **TanStack Query** | Optimistic updates, cache management. |
| Mobile | **React Native (Expo)** | Phase 7. Reuses ~60% of web business logic. |
| Hosting | **Railway / Render** | Cheap production hosting (~KES 5-7K/month). |
| File storage | **Cloudflare R2** | S3-compatible, no egress fees, much cheaper than AWS. |
| Integrations | M-PESA Daraja · Africa's Talking SMS · Resend email | Kenyan-friendly providers. |

## Multi-tenancy strategy

- **Single database, shared schema, row-level filtering by `organizationId`.**
- Every tenant-scoped model has an `organizationId` foreign key.
- PostgreSQL **Row-Level Security (RLS)** policies enforce isolation at the database level — not relying on app-layer filters alone.
- A `shared` flag on Donor and Partner allows a single record to surface across orgs that share that relationship.
- Per-org branding tokens (colour, logo, name) loaded from the Organization table at app boot.

## RBAC — default roles

| Role | Permissions |
|---|---|
| **Admin** | Full read/write within their organization(s) |
| **Programme Manager** | Read/write programme & beneficiary data; read-only finance |
| **Finance** | Full finance access; read-only beneficiary data |
| **Communications** | Full bulk SMS/email access; read-only people |
| **Field Staff** | Mobile-app data capture; read-only office data |
| **Board Observer** | Read-only dashboards across all programmes |

A user can hold different roles in different organizations (e.g., Admin at Darajani, Field Staff at Agape).

## Getting started

```bash
# 1. Install dependencies
npm install

# 2. Set up environment
cp .env.example .env
# Edit .env with DATABASE_URL, NEXTAUTH_SECRET, etc.

# 3. Run database migrations
npx prisma migrate dev --name init

# 4. Seed initial data (3 orgs + roles + sample admins)
npx prisma db seed

# 5. Start dev server
npm run dev
```
